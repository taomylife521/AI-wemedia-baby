# Antigravity Agent 核心行为准则 (Strict Chinese Directive)

## 1. 语言一致性极高要求 (Language Absolute)

- **强制指令**：禁止以任何理由在输出中使用英文（代码语法关键词除外）。
- **全流程汉化**：
  - 所有的思考过程 (Thought)、步骤拆解 (Plan)、任务列表 (Task) 必须使用简体中文。
  - 所有的进度反馈（如 "Analyzing", "Editing"）必须在输出时即时翻译为“正在分析”、“正在编辑”。
- **禁止英文模板**：即使系统底层提供了英文模板（如 implementation_plan.md），你必须将其表头、说明、状态全部翻译成中文后再展示。

## 2. 开发者友好输出 (Developer UI/UX)

- **代码注释与标注**：
  - 生成的代码中，所有 UI 控件（按钮、标签、弹窗）的 `text` 属性必须直接硬编码为中文。
  - 所有新增或修改的代码必须包含详细的中文注释。
- **报错处理**：如果遇到编译器或系统报错，必须先翻译报错含义，再用中文给出解决方案。

## 3. Gemini 模型特定增强 (Gemini Optimization)

- **思考链强制**：在每一步 `Thought` 的开头，必须包含“【思考】”前缀，并确保其内容完全不含英文单词。
- **任务状态**：将 `[Todo]`, `[Done]`, `[In Progress]` 替换为 `[待办]`, `[完成]`, `[进行中]`。

## 4. 交互逻辑

- **简洁性**：遵循 KISS 原则，解释要通俗易懂，避开深奥的英文术语。
- **纠错准则**：如果发现用户指令有误，请用中文温和地指出并提供修正建议。
