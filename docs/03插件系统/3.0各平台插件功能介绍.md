# 3.0 插件系统架构与各平台插件功能介绍

## 1. 插件系统概述

媒小宝采用**插件化架构**来管理各社交平台的自动化登录验证与内容发布能力。所有插件相关的代码统一存放在 `src/plugins/` 目录下，按照**框架引擎**、**社区插件**和**专业插件**三个层级进行组织管理。插件只为平台提供"登录验证"和"内容发布"这两个原子化的自动化能力。

## 3. 核心接口

### 3.1 登录插件接口 (`LoginPluginInterface`)

所有平台的登录插件都必须实现以下抽象方法：

| 方法                             | 返回值        | 说明                                                                       |
| -------------------------------- | ------------- | -------------------------------------------------------------------------- |
| `check_login_status(context)`    | `bool`        | 在浏览器环境中通过注入脚本检测当前页面是否处于登录态                       |
| `extract_user_info(context)`     | `LoginResult` | 提取当前已登录用户的昵称、头像等信息及全部 Cookies                         |
| `verify_account_status(context)` | `LoginResult` | 综合验证方法，优先尝试 HTTP 接口校验 Cookie 有效性，失败后降级到浏览器验证 |

### 3.2 发布插件接口 (`PublishPluginInterface`)

所有平台的发布插件都必须实现以下抽象方法：

| 方法                                    | 返回值            | 说明                                                                 |
| --------------------------------------- | ----------------- | -------------------------------------------------------------------- |
| `get_form_schema(content_type)`         | `List[FormField]` | 返回该平台发布表单的字段定义（标题、描述、标签等），用于 UI 动态渲染 |
| `publish(context, file_path, metadata)` | `PublishResult`   | 执行完整的自动化发布流程：上传素材 → 填写表单 → 提交确认             |

## 4. 插件管理器 (`PluginManager`)

`PluginManager` 位于 `src/plugins/core/plugin_manager.py`，负责在应用启动时自动扫描并注册所有可用插件：

1. **社区插件扫描**：遍历 `src/plugins/community/` 下的每个平台子目录，寻找 `login_plugin.py` 和 `publish_plugin.py`，通过 `importlib` 动态加载并实例化。
2. **专业插件扫描**：若 `src/plugins/pro/platforms/` 目录存在，则以同样的方式加载闭源平台插件。
3. **按需获取**：业务代码通过 `PluginManager.get_login_plugin("douyin")` 或 `PluginManager.get_publish_plugin("kuaishou")` 按平台名称获取对应的插件实例。

## 5. 各平台插件一览

| 平台           | 分类 | 登录插件 | 发布插件 | 特点                              |
| -------------- | ---- | -------- | -------- | --------------------------------- |
| **抖音**       | 社区 | ✅       | ✅       | 支持创作者 API + 浏览器双通道验证 |
| **快手**       | 社区 | ✅       | ✅       | 支持页面脚本注入检测登录态        |
| **微信视频号** | 专业 | ✅       | ✅       | 基于复杂扫码和多标签防检测策略    |
| **小红书**     | 专业 | ✅       | 开发中   | 使用 HTTP 接口静默验证 Cookie     |

## 6. 与专业版功能模块的关系

| 模块      | 存放位置                        | 性质                                      |
| --------- | ------------------------------- | ----------------------------------------- |
| 平台插件  | `src/plugins/`                  | 实现标准接口的可替换自动化模块            |
| 批量发布  | `src/pro_features/batch/`       | 独立页面功能（含 pages/services/ui 三层） |
| 数据中心  | `src/pro_features/data_center/` | 独立页面功能                              |
| 评论/私信 | `src/pro_features/interaction/` | 独立页面功能                              |

插件系统仅负责提供"登录"和"发布"两个原子能力；而批量发布等高级功能则在上层调用插件的能力来完成自身的业务逻辑。
