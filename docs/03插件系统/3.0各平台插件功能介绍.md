# 3.0 各平台插件功能介绍

> **文档说明**：本节介绍插件系统整体架构、核心接口、目录约定及各平台插件一览。单平台实现细节见 [3.1 抖音插件功能介绍](3.1抖音插件功能介绍.md)；防风控机制见 [防风控机制设计](防风控机制设计.md)。

---

## 1. 插件系统概述

媒小宝采用**插件化架构**管理各社交平台的**登录验证**与**内容发布**能力。插件仅提供这两类原子能力，批量发布、发布列表等业务由主程序调用插件完成。

- **代码位置**：所有插件相关代码位于 `src/plugins/`。
- **分层**：**框架引擎**（core）、**社区插件**（community）、**专业插件**（pro）。
- **扩展方式**：新增平台时在 `community/` 或 `pro/` 下新增平台目录，实现标准接口即可被自动扫描加载。

---

## 2. 目录结构

```text
src/plugins/
├── __init__.py                    # 对外暴露 PluginManager
├── core/                          # 插件框架引擎
│   ├── plugin_manager.py         # 插件加载器（扫描 community/、pro/）
│   └── interfaces/
│       ├── login_plugin.py       # LoginPluginInterface、LoginResult、AccountVerificationContext
│       └── publish_plugin.py     # PublishPluginInterface、PublishResult、FormField
├── community/                     # 社区插件（开源，随主仓库发布）
│   ├── douyin/                    # 抖音
│   │   ├── login_plugin.py
│   │   ├── publish_plugin.py
│   │   ├── scripts.py
│   │   ├── selectors.py
│   │   ├── config.json
│   │   └── steps/                 # 发布步骤链（step_01～step_08、step_runner）
│   └── kuaishou/                  # 快手
│       ├── login_plugin.py
│       ├── publish_plugin.py
│       └── scripts.py
└── pro/                           # 专业插件（若存在则加载，可被 .gitignore）
    └── <平台名>/                  # 每平台一目录，与 community 下结构约定一致
        ├── login_plugin.py
        ├── publish_plugin.py
        └── …
```

- **加载规则**：`PluginManager` 遍历 `community/` 与 `pro/` 的**一级子目录**，对每个子目录尝试加载 `login_plugin.py`、`publish_plugin.py` 中实现对应接口的类并注册到 `platform_id`。
- **抖音**：当前实现最完整，含 8 步发布链、防风控对接、选择器与步骤文件，可作为其他平台的**设计规范参考**（实现独立，不跨平台复用代码）。

## 3. 核心接口

### 3.1 登录插件接口 (`LoginPluginInterface`)

| 方法                             | 返回值        | 说明                                                    |
| -------------------------------- | ------------- | ------------------------------------------------------- |
| `platform_id`                    | `str`         | 平台标识，如 `"douyin"`、`"kuaishou"`                   |
| `platform_name`                  | `str`         | 显示名称，如「抖音」「快手」                            |
| `login_url`                      | `str`         | 创作者中心登录页 URL                                    |
| `check_login_status(context)`    | `bool`        | 在浏览器环境中检测当前页面是否已登录                    |
| `extract_user_info(context)`     | `LoginResult` | 提取昵称、头像、Cookies 等                              |
| `verify_account_status(context)` | `LoginResult` | 综合验证（优先 HTTP 校验 Cookie，失败则降级浏览器验证） |

**LoginResult**：`success`、`cookies`、`nickname`、`avatar_url`、`user_id`、`error_message`（掉线原因，主程序用于任务失败展示）等。

### 3.2 发布插件接口 (`PublishPluginInterface`)

| 方法                                    | 返回值            | 说明                                                   |
| --------------------------------------- | ----------------- | ------------------------------------------------------ |
| `platform_id`                           | `str`             | 平台标识                                               |
| `get_form_schema(content_type)`         | `List[FormField]` | 发布表单字段定义（标题、描述、标签等），供 UI 动态渲染 |
| `publish(context, file_path, metadata)` | `PublishResult`   | 执行完整发布流程：上传 → 填表 → 提交                   |

**PublishResult**：`success`、`publish_url`、`error_message`、`failed_step`（失败时步骤名，便于主程序/UI 展示）。

**说明**：`context` 为主程序传入的 Playwright `Page`（或浏览器上下文），已具备账号 Cookie 及浏览器层防风控环境；`metadata` 含标题、描述、标签、`file_type`、`speed_rate` 等。

## 4. 插件管理器 (`PluginManager`)

- **位置**：`src/plugins/core/plugin_manager.py`
- **初始化**：应用启动时调用 `PluginManager.initialize()`，扫描 `community/` 与 `pro/` 并加载插件。
- **获取插件**：
  - `PluginManager.get_login_plugin(platform_id)` → 登录插件实例
  - `PluginManager.get_publish_plugin(platform_id)` → 发布插件实例
- **可用平台**：`PluginManager.get_available_platforms()` 返回已加载的 `platform_id` 列表（以登录插件为准）。

## 6. 与主程序及其他模块的关系

- **主程序发布流程**：发布列表/批量发布等触发任务后，主程序「刷新账号登录状态」时调用对应平台的**登录插件**（如 `verify_account_status`），「开始发布任务」时调用**发布插件**的 `publish(context, file_path, metadata)`。插件返回 `PublishResult` 或登录结果，主程序据此更新任务状态与日志。
- **防风控**：浏览器层（Undetected Playwright、指纹、stealth）由主程序/基础设施在打开浏览器时配置；发布层（延迟、拟人点击/输入、步骤间间隔、操作频繁冷却）由公共模块 `src/infrastructure/anti_risk/` 提供，各平台发布插件在步骤中调用。详见 [防风控机制设计](防风控机制设计.md)。
- **专业版功能**：批量发布、数据中心、评论私信等为独立功能模块，通过调用 `PluginManager` 获取登录/发布插件完成能力，插件系统本身不包含这些业务逻辑。

## 7. 扩展新平台时的设计规范（参考抖音）

新平台（如快手、小红书）**独立实现**登录与发布插件，不复用抖音代码，但建议与抖音插件保持**同一设计规范**，便于维护与统一体验：

- **目录**：`src/plugins/community/<平台_id>/` 或 `pro/<平台_id>/`，内含 `login_plugin.py`、`publish_plugin.py` 及平台所需 `selectors`、`scripts`、配置等。
- **发布流程**：可采用与抖音类似的「步骤链 + StepRunner」结构（每步返回 继续/结束/补救），或单块流程；关键操作建议调用公共 anti_risk（`human_click`、`human_type_text`、`step_interval`、`random_delay` 等）。
- **配置**：平台级配置（如 `anti_risk`、limits）放在 `config/platforms/<平台_id>.json`，由插件在 `publish()` 入口读取并传入步骤或流程。
- **日志**：用户可见的发布日志建议统一通过 `logging.getLogger("publish.user_log")` 输出，便于主程序发布日志框展示。

具体步骤划分、文件命名、防风控对接方式可参考 [3.1 抖音插件功能介绍](3.1抖音插件功能介绍.md) 与 [防风控机制设计](防风控机制设计.md)。
