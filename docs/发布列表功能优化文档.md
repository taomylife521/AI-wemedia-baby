# 发布列表功能优化文档

## 1. 业务流程概述

本模块主要负责自动化监控与执行用户的媒体发布任务，其核心设计理念是**“调度与执行分离”**。统筹任务的派发和状态管理由发布列表模块负责，而具体到各个平台（如抖音、快手、小红书等）的自动化操作细节，则完全下放给对应平台的独立插件。

整体业务流转如下：

1. **任务触发与取单**：用户在发布列表页面点击“开始”按钮后，系统扫描任务列表，提取排在第一位的待执行任务。
2. **环境准备与加载**：系统解析该任务对应的平台及账号名称，拉起对应的浏览器进程（使用持久化上下文或加载有效 Cookie），并导航到该平台的特定发布入口页面。
3. **插件调用与发布**：
   - 调度中心通过插件管理器，调用与目标平台对应的高级发布插件（如 `DouyinPublishPlugin`）。
   - 插件接管页面控制权，执行文件上传、标题/描述填写、标签添加等专有的 DOM 操作流程。
4. **结果回收与状态流转**：
   - 插件执行完毕后，将标准化的执行结果（成功/失败原因及返回的URL等）传递给发布列表调度模块。
   - 调度模块将结果同步至 UI 层。
     - **发布成功**：UI 摘除当前任务，将其移动或标记至“发布记录”归档页面。
     - **发布失败**：UI 驻留该任务，并将状态及错误日志反馈在界面上，便于用户审查或重试。

---

## 2. 调度执行分离机制

### 2.1 调度模块（Publish Executor & UI）

发布列表模块作为核心中控，其主要职责限定于宏观调度：

- **队列管理**：管理发布任务的执行顺序（单步、批量）以及启停控制。
- **环境隔离**：负责调起关联账号对应的数据目录，确保防关联浏览器环境独立、安全。
- **生命周期对接**：封装标准接口请求所持有的目标参数结构，唤起指定插件并保持通信、捕获超时异常，完成后收回浏览器管控权进行后处理资源清理。
- **UI 呈现**：仅对发布的总状态（如处理中、失败、成功）进行数据层的修改及前端渲染，并不处理实际页面自动化。

### 2.2 平台插件模块（Platform Plugins）

各平台发布插件作为动作执行者，承载实际发布流程中的一切脏活累活。

- **单一职责**：只专注于如何在对应的平台网页（或接口）完成一套标准的发布行为，无须关心任务如何轮转。
- **差异抹平**：解决各平台由于UI结构突变、拦截校验引发的异常。统一向上返回 `PublishResult(success=True/False)` 标准模型数据。
- **灵活热拔**：如果任何平台规则发生变动，只需要更新其针对的插件文件实现，发布调度与UI逻辑零干扰。

---

## 3. 核心优势与优化方向

- **低耦合、高内聚**：彻底杜绝控制层代码中杂糅大量枯燥重复的 `page.locator()` 定位器，保持中台系统清洁。
- **可扩展性强**：横向接入新平台（如 B站、视频号）时，仅需继承标准插件接口实现一遍“登录验证”和“发推代码”即可快速上线，且不影响其它平台稳定性。
- **错误排查清晰**：发布链路的报错隔离在了单独的平台插件执行域内，只需定位如 `douyin/publish_plugin` 即可解决相应环境的适配问题。
