# 抖音自动发布功能技术方案

## 1. 方案选择：API 还是 浏览器自动化 (Playwright)？

在开发抖音发布功能时，主要有两种技术路线：

### 方案 A：纯 API 逆向 (HTTP 请求)

- **优点**：资源占用低，速度快，适合大规模并发。
- **缺点**：
  - **签名复杂**：抖音 API 使用 `X-Bogus`, `msToken`, `ttwid` 等复杂的签名算法，且算法经常变动。
  - **风控严格**：纯 API 请求容易触发验证码或封号。
  - **维护成本高**：每次抖音前端更新，都需要重新逆向。

### 方案 B：浏览器自动化 (Playwright)

- **优点**：
  - **稳定性极高**：模拟真实用户操作，自动处理 JS 执行和签名生成。
  - **开发快**：无需逆向复杂的加密算法。
  - **抗风控**：配合指纹混淆技术，更像真人操作。
- **缺点**：资源占用稍高（需要运行浏览器内核）。

### 结论

鉴于本项目 `wemedia-baby` 旨在提供稳定可靠的桌面客户端，且主要服务于中小规模账号管理，**强烈推荐使用 方案 B (Playwright)**。
同时，利用 `自媒体平台分析软件` 的抓包结果作为参考，辅助定位关键 CSS 选择器和理解后台逻辑。

### 2.2 流程状态机

发布任务应包含以下原子状态，以支持断点续传和错误恢复：

1. `INIT`: 初始化，检查文件存在性。
2. `LOGIN_CHECK`: 检查 Cookie 有效性。
3. `NAVIGATE`: 跳转至发布页。
4. `UPLOADING`: 正在上传视频（需要监听进度）。
5. `PROCESSING`: 视频转码/检测中。
6. `FILLING`: 填写标题、话题、封面。
7. `SUBMITTING`: 点击发布按钮。
8. `VERIFYING`: 确认发布成功（检测跳转或成功提示）。

## 3. 详细实施步骤 (Playwright)

### 3.1 页面导航与检测

- **目标 URL**: `https://creator.douyin.com/creator-micro/content/upload`
- **检查点**:
  - 等待 `input[type="file"]` 出现表示加载成功。
  - 如果跳转到 `/login`，触发异常，要求重新登录。

### 3.2 视频上传 (难点突破)

抖音上传组件通常是隐藏的 `<input type="file">`。

```python
# 核心代码逻辑
async def upload_video(page: Page, file_path: str):
    # 1. 设置文件选择拦截器
    async with page.expect_file_chooser() as fc_info:
        # 点击触发上传区域（或者直接定位 input）
        await page.locator("div.upload-btn").click()
    file_chooser = await fc_info.value
    await file_chooser.set_files(file_path)

    # 2. 监听上传进度
    # 策略：检测界面上的进度条文本 "上传中 50%" 或 "上传成功"
    await page.wait_for_selector("text=上传成功", timeout=60000)
```

### 3.3 封面设置 (可选)

如果需要自定义封面，需要处理 Canvas 裁剪或上传图片。

- **推荐策略**：V1 版本先默认使用视频第一帧，V2 版本增加自定义封面支持（需要处理弹窗）。

### 3.4 话题与艾特

抖音的输入框是一个 `contenteditable` 的 `div`，而不是简单的 `input`。

- **难点**：直接 `fill` 可能不生效。
- **解决方案**：使用 `type` 模拟人工输入。
  - 输入 `#` 自动触发话题下拉框。
  - 输入 `@` 自动触发用户列表。
  - 需要 `await page.keyboard.press("Enter")` 确认选择。

### 3.5 发布设置

- **位置**: 点击“添加位置” -> 输入城市 -> 选择下拉项。
- **隐私设置**: 公开/好友可见/私密 (通过 Radio Button 选择)。
- **发布时间**: 立即发布 vs 定时发布 (DatePicker 处理)。

### 3.6 提交与确认

- **点击发布**: `button:has-text("发布")`。
- **成功判定**:
  - 监听 URL 变化：通常跳转到 `contents/manage`。
  - 监听 Toast 提示：`div:has-text("发布成功")`。

## 4. 异常处理机制

| 异常场景       | 表现                       | 处理策略                                                  |
| :------------- | :------------------------- | :-------------------------------------------------------- |
| **Cookie失效** | 页面重定向至 login         | 抛出 `AuthError`，任务状态置为 `FAILED`，通知用户重新登录 |
| **上传卡死**   | 进度条长时间不动           | 设置 3 分钟超时重试，重试 3 次后失败                      |
| **版权拦截**   | 上传后提示“版权风险”       | 截图保存证据，标记任务为 `BLOCKED`                        |
| **每日上限**   | 提示“今日发布次数已达上限” | 解析提示中的恢复时间，自动设置定时任务延后执行            |

## 5. 开发建议

1. **利用分析工具**: 虽然我们不直接用 API 发包，但可以使用您目录下的 `自媒体平台分析软件` 的 `browser` 模块，复用其 **CSS 选择器配置** (`selectors.json`)，这样当抖音改版时，只需更新配置文件，不用改代码。
2. **调试模式**: 开发阶段设置 `headless=False`，肉眼观察机器人操作。
3. **指纹防护**: 必须注入 `stealth.min.js`，防止被识别为 WebDriver。

## 6. 总结

不需要完全重写爬虫。直接利用 `自媒体平台分析软件` 对**元素定位**和**流程逻辑**的分析结果，在 `wemedia-baby` 中用 Playwright 实现自动化操作。这是最稳妥的路线。
