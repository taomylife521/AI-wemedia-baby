## 1. 功能概述

### 1.1 需求背景

当前系统的账号管理功能已经满足基本的账号登录和状态管理需求，但在多平台账号发布场景下存在以下痛点：

1. **发布效率低下**：用户需要为每个平台单独选择账号，无法一键选择多平台账号进行统一发布
2. **账号组织混乱**：随着账号数量增加，缺乏有效的分组管理机制
3. **跨平台发布复杂**：同一内容发布到多个平台时，需要重复操作

### 1.2 目标功能

引入**账号库**一级菜单，包含两个核心子功能：

| 子功能         | 说明                                   |
| -------------- | -------------------------------------- |
| **账号管理**   | 原有功能的升级版，新增"账号组"列       |
| **账号组管理** | 新增功能，支持将多平台账号组合成账号组 |

### 1.3 核心概念：账号组

**账号组**是将**不同平台**的多个账号组合在一起的逻辑单元：

- 每个账号组可包含多个账号，但**同一平台最多只能有一个账号**
- 新登录的账号默认状态为"未分组"
- 发布时可选择账号组，系统自动向组内所有账号发布内容

```
账号组示例：
┌─────────────────────────────────────┐
│  账号组: "个人矩阵1"                  │
│  ├── 抖音: @张三_抖音                 │
│  ├── 快手: @张三_快手                 │
│  └── 小红书: @张三的小红书            │
└─────────────────────────────────────┘
```

## 3. 数据库设计

### 3.1 新增表：account_groups

```sql
CREATE TABLE IF NOT EXISTS account_groups (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    user_id INTEGER NOT NULL,
    group_name TEXT NOT NULL,           -- 账号组名称
    description TEXT,                    -- 描述（可选）
    created_at TEXT DEFAULT CURRENT_TIMESTAMP,
    updated_at TEXT,
    FOREIGN KEY (user_id) REFERENCES users(id)
);

CREATE INDEX IF NOT EXISTS idx_account_groups_user_id ON account_groups(user_id);
CREATE UNIQUE INDEX IF NOT EXISTS idx_account_groups_name ON account_groups(user_id, group_name);
```

### 3.2 修改表：platform_accounts

添加 `group_id` 外键字段：

```sql
-- 新增字段
ALTER TABLE platform_accounts ADD COLUMN group_id INTEGER REFERENCES account_groups(id);

-- 新增索引
CREATE INDEX IF NOT EXISTS idx_platform_accounts_group_id ON platform_accounts(group_id);
```

### 3.3 数据关系

```
account_groups (1) ────< (N) platform_accounts

约束条件：
- 一个账号组可以包含多个账号
- 一个账号只能属于一个账号组（或未分组）
- 同一账号组内，每个平台最多只能有一个账号
```

class AccountGroup:
    """账号组领域实体"""
    group_id: Optional[int] = None
    user_id: int
    group_name: str
    description: Optional[str] = None
    created_at: datetime = field(default_factory=datetime.now)

## 7. 风险与注意事项

| 风险                   | 缓解措施                                     |
| ---------------------- | -------------------------------------------- |
| 数据库迁移失败         | 添加迁移前备份逻辑                           |
| 现有发布逻辑兼容性     | 保持 AccountSelectionDialog 返回格式向后兼容 |
| 账号组内同平台限制     | 在服务层添加强校验，UI 层给予明确提示        |
| 性能问题（大量账号组） | 分页加载、虚拟列表渲染                       |
