# 刷新登录状态功能介绍与优化方案

## 1. 功能概述

"刷新登录状态"用于判断已登录账号当前的登录凭证（Cookie）是否仍然有效，从而确定账号为**在线**还是**离线**状态。

刷新登录状态本身是一个**独立模块**（`AccountVerifier` + `AccountValidatorService`），它不包含任何平台特有的验证逻辑。具体判断"某个平台的 Cookie 是否有效"的能力由该平台对应的**插件**（`LoginPluginInterface.verify_account_status`）提供。

---

## 2. 触发入口

### 2.1 批量刷新（账号管理界面按钮）

- **位置**：账号管理页面顶部工具栏的"刷新登录状态"按钮
- **范围**：对账号列表中**所有已登录账号**执行登录状态验证
- **执行模式**：**多线程/多协程并发模式**
  - 先按平台对账号分组
  - 对每个平台下的账号，使用 `asyncio.Semaphore` 控制并发（默认最大 5 个并发）
  - 每个验证任务之间添加随机延迟（0.5~1.5 秒），避免高频请求被平台限流
- **UI 表现**：弹出进度条对话框，实时显示"已验证 X / 共 Y 个"

### 2.2 单账号刷新（右键菜单）

- **位置**：右键点击账号列表中的某一行 → 弹出菜单 → "刷新登录状态"
- **范围**：仅针对**选中的单个账号**
- **执行模式**：**单线程/单任务模式**
  - 直接判断该账号所属平台
  - 调用该平台插件的验证逻辑
  - 等待结果返回后更新 UI
- **UI 表现**：静默执行，完成后直接刷新该行的状态图标，无弹窗

## 5. 现存问题与优化建议

### 5.1 ✅ 按平台分组的并发调度（已实施）

**现状**：当前批量刷新使用 `asyncio.Semaphore(5)` 做全局并发控制。5 个并发槽位不区分平台，可能出现 5 个任务全在同一平台排队，而其他平台空闲的情况。

**优化建议**：引入**按平台分组调度**策略：

```python
# 优化后伪代码
platform_groups = group_accounts_by_platform(accounts)
for platform, group_accounts in platform_groups.items():
    # 每个平台独立的信号量（如每平台最多 3 个并发）
    platform_semaphore = asyncio.Semaphore(3)
    tasks = [verify_with_semaphore(platform_semaphore, acc) for acc in group_accounts]
    # 所有平台的任务一起并发执行
    all_tasks.extend(tasks)
await asyncio.gather(*all_tasks)
```

这样同一平台的请求不会互相拥堵，不同平台之间真正实现了并行。

### 5.2 ✅ 逐条实时刷新（已实施）

**现状**：批量刷新完成后，UI 收到的是全量结果，然后一次性刷新整个列表。在账号数量多（如 50+）时，用户需要等到全部完成才能看到结果。

**优化建议**：实现**逐条实时刷新**：

- `AccountVerifier` 每完成一个账号就通过 `progress_callback` 返回该账号验证结果
- `AccountValidatorService` 收到后立即发射 `progress` 信号
- UI 层收到信号后**立即更新该行的状态图标**，而不是等到全部结束

### 5.3 ✅ 离线原因悬浮提示（已实施）

**现状**：账号被判定为离线后，用户无法在界面上看到具体原因（是 Cookie 过期？Cookie 文件丢失？还是网络超时？）。

**优化建议**：在账号列表的"状态"列增加**悬浮提示**（Tooltip），展示最近一次验证的 `error` 信息。数据来源于 `AccountVerifier` 返回的 `result['error']` 字段。

### 5.4 ✅ 验证超时保护（已实施）

**现状**：`aiohttp.ClientTimeout(total=10)` 设置了 10 秒超时，但某些平台的验证接口在网络波动时可能长时间无响应，导致所有并发槽被占满。

**优化建议**：为每个单独的验证任务增加 `asyncio.wait_for` 包装：

```python
try:
    result = await asyncio.wait_for(
        plugin.verify_account_status(context),
        timeout=15  # 15秒硬超时
    )
except asyncio.TimeoutError:
    result = LoginResult(success=False, error_message="验证超时")
```

### 5.5 定时自动静默刷新

**现状**：刷新登录状态必须由用户手动触发。

**优化建议**：可在设置页面增加"自动刷新间隔"选项（如每 30 分钟/1 小时/关闭），由后台定时任务以静默模式（`silent=True`）调用 `start_verify_all()`，自动保持账号状态为最新，无需用户干预。
