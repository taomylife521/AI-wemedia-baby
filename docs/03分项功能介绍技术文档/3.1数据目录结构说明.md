# 数据目录结构说明文档

> **文档版本**：v1.1  
> **最后更新**：2026-03-01

## 1. 概述 (Overview)

为了更好支持多平台、多账号管理，本项目采用了 **"Account-Centric" (以账号为中心)** 的数据存储结构。所有的账号数据（包括浏览器缓存、指纹配置、工作区文件、Cookie备份等）均统一归档在以 `平台 > 平台账号(platform_username)` 命名的目录下。

这种结构确数据的**高内聚性**和**易管理性**，便于未来的数据迁移、备份及排查问题。

## 2. 目录树结构 (Directory Tree)

所有应用数据存储在 `%LOCALAPPDATA%/WeMediaBaby/data` (Windows) 或 `~/.WeMediaBaby/data` (Mac/Linux) 下。

```text
WeMediaBaby/
└── data/
    ├── (密钥已移除: 转存至 Windows Credential Manager)
    │
    ├── douyin/                       # [平台 层级] (例如: douyin, kuaishou)
    │   └── my_douyin_id/             # [账号 层级] (使用 platform_username, 如抖音号/自媒体ID)
    │       │
    │       ├── browser/              # [Browser] Playwright 浏览器数据
    │       │   ├── Default/          # Chrome 用户配置文件
    │       │   ├── storage_state.json # 自动登录凭证 (Playwright)
    │       │   ├── fingerprint_config.json # 浏览器指纹配置
    │       │   └── user_data/        # 用户数据目录
    │       │
    │       ├── qt_profile/           # [Browser] QtWebEngine 浏览器数据 (混合模式用)
    │       │   ├── cache/            # 缓存文件
    │       │   └── ...
    │       │
    │       ├── workspace/            # [Workspace] 账号工作区
    │       │   ├── media/            # 素材管理
    │       │   │   ├── pending/      # 待发布
    │       │   │   └── published/    # 已发布
    │       │   ├── logs/             # 账号专属日志
    │       │   └── temp/             # 临时文件
    │       │
    │       └── backup.encrypted      # [Backup] 加密的 Cookie/账号信息备份
    │
    └── kuaishou/                     # 其他平台...
        └── ...
```

## 3. 各目录/文件用途详解

| 路径/文件                             | 类型 | 描述                                                                                                            |
| :------------------------------------ | :--- | :-------------------------------------------------------------------------------------------------------------- |
| `data/{platform}/`                    | 目录 | 平台根目录，如 `douyin`, `xiaohongshu`。                                                                        |
| `.../{platform_username}/`            | 目录 | 账号根目录。必须使用 **平台唯一标识符** (platform_username)，确保唯一性且不可变。                               |
| `.../browser/`                        | 目录 | **Playwright** 浏览器的 `user_data_dir`。包含 Cookies, LocalStorage, History 等。**删除此目录会导致登录失效。** |
| `.../browser/storage_state.json`      | 文件 | Playwright 的登录状态快照，用于免登录启动。                                                                     |
| `.../browser/fingerprint_config.json` | 文件 | 该账号绑定的浏览器指纹配置（UserAgent, Canvas噪声等），确保环境一致性。                                         |
| `.../qt_profile/`                     | 目录 | **QtWebEngine** 的持久化存储路径。当使用内嵌浏览器模式时，数据存储于此。                                        |
| `.../workspace/`                      | 目录 | 账号的业务数据工作区。                                                                                          |
| `.../backup.encrypted`                | 文件 | 使用 `Fernet` 加密的 Cookie 和账号元数据备份。用于在浏览器数据丢失时恢复，或跨设备迁移。                        |

## 4. 技术实现 (Technical Implementation)

### 4.1 核心路径管理器 (`PathManager`)

所有路径的生成统一由 `src/infrastructure/common/path_manager.py` 管理，严禁在业务代码中手动拼接字符串路径。

**API 使用示例:**

```python
from src.infrastructure.common.path_manager import PathManager

# 获取某账号的根目录 (会自动创建目录)
# 结果: .../data/douyin/media_expert_01
account_root = PathManager.get_platform_account_dir(
    platform="douyin",
    platform_username="media_expert_01"
)

# 基于根目录派生子路径
workspace = account_root / "workspace"
browser_dir = account_root / "browser"
```

### 4.2 浏览器配置管理 (`ProfileManager`)

`src/infrastructure/browser/profile_manager.py` 负责管理具体的文件路径。

- **初始化**: 需要传入 `platform` 和 `platform_username`。
- **功能**:
  - `get_user_data_dir()`: 获取 Playwright 数据目录。
  - `get_fingerprint()`: 读取或生成指纹配置。

### 4.3 兼容性说明

为了兼容旧版代码，部分组件仍保留了仅基于 `account_id` (数字ID) 的初始化方式，但**已被强制重定向**到 `%LOCALAPPDATA%` 下的统一目录，不再回退到 `%APPDATA%` (Roaming)。

**新开发功能必须使用包含 platform 信息的 API 以启用 Account-Centric 结构。**

### 4.5 缓存管理 (Cache Management)

系统实现了递归缓存清理机制 (`CacheManager`)，覆盖以下路径：

- **全局日志**: `WeMediaBaby/logs/`
- **QtWebEngine 缓存**: `.../qt_profile/cache`, `Code Cache`, `GPUCache` 等。
- **工作区临时文件**: `.../workspace/temp/`
- **工作区日志**: `.../workspace/logs/`

清理操作通过“设置 -> 数据管理 -> 清理缓存”触发。

## 5. 开发规范

1.  **新建账号**: 使用 `AccountManagerAsync.add_account`，它会自动调用 `PathManager` 创建标准目录结构。
2.  **文件读写**: 始终使用 `PathManager` 获取基准路径。
3.  **删除账号**: 删除账号在数据库的记录时，应询问用户是否同步删除 `data/{platform}/{platform_username}` 下的物理文件。

### 4.4 安全机制 (Security)

敏感数据（如 Cookie 加密密钥）不再落地存储于文件的 `keys/` 目录，而是使用系统级 **Keyring** (Windows Credential Manager) 进行安全存储。
服务名称: `WeMediaBaby`
用户名格式: `key_{user_id}`
