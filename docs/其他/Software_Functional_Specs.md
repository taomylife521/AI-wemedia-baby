## 2. 功能模块详述

### 2.1 工作台 (Dashboard)

**功能目标**：提供全局概览和快捷入口。

- **数据概览**：展示今日发布数、成功率、账号在线总数。
- **快捷入口**：提供“新建发布”、“添加账号”的快速按钮。
- **系统通知**：显示软件更新、会员到期等重要通知。

### 2.2 账号管理 (Account Management)

**核心功能**：账号的全生命周期管理与状态监控。

#### 2.2.1 添加账号 (Add Account)

- **技术方案**：**Undetected Playwright (Headed Mode)**
- **交互流程**：
  1.  点击“添加账号” -> 选择平台（抖音/快手）。
  2.  系统启动**有头**浏览器窗口，自动跳转至登录页。
  3.  用户手动完成扫码/验证。
  4.  系统监听登录成功状态，自动保存凭证 (`storage_state.json`) 及指纹配置 (`fingerprint_config.json`)。
  5.  自动关闭验证窗口并刷新列表。

#### 2.2.2 账号列表与监控

- **状态监控**：支持手动点击“刷新状态”，后台快速检测 Cookie 有效性（在线/失效/风险）。
- **浏览器操作 (Inspection Mode)**：
  - **快速访问**：双击账号条目，以**有头模式**启动绑定了环境与凭证的浏览器，自动免登进入后台。
  - **批量开启**：勾选多个账号 -> 点击“打开浏览器”，并发启动多个独立环境（用于人工多开养号）。
- **数据隔离**：所有数据存储于 `%APPDATA%/WeMediaBaby/data/browsers/{account_id}/`，实现物理级环境隔离。

### 2.3 发布管理 (Publish Management)

**功能目标**：任务队列的监控与干预，支持高效并发发布。

- **浏览器模式控制**：
  - **显示浏览器界面 (复选框)**：
    - **未勾选（默认）**：采用 **Headless (无头)** 模式执行任务，节省系统资源。
    - **已勾选**：采用 **Headed (有头)** 模式执行，用户可实时查看浏览器操作细节。
  - **执行中切换**：若任务正在以无头模式运行，用户可在发布列表中点击“显示窗口”按钮，动态切换至有头模式，方便随时观察发布过程及进度或进行人工干预。
- **并发策略**：
  - **多线程支持**：系统支持手动设置发布线程数，最大支持 **3 线程** 并发。
  - **提速机制**：开启多线程后，系统将同时调度多个不同账号执行任务，显著缩短整体耗时。
  - **账号互斥原则**：**同一账号不支持多线程并发**。系统确保单个账号在同一时刻仅有一个任务运行，以规避平台风控风险。
- **发布列表**：显示 Pending/Running 任务。支持暂停、取消、插队、状态实时监控。
- **发布记录**：历史任务查询（支持按平台/账号/时间筛选），支持“一键重发”失败任务。

#### 2.3.1 核心策略

- **浏览器模式控制**：
  - **显示界面 (Headed)**：勾选“显示浏览器窗口”复选框后，发布任务将以**有头模式**启动，用户可实时通过 UI 观察执行细节。
  - **动态切换**：支持在任务运行中途，通过点击“显示窗口”按钮，将不可见的无头实例切换为可见的有头实例（需技术验证支持）或在下一次任务启动时生效。默认推荐**无头模式**以节省资源。
- **并发策略**：
  - **多线程并发**：支持用户自定义并发数（默认最大 3 线程），允许 3 个**不同账号**同时执行发布。
  - **账号互斥原则**：**同一账号**在同一时刻仅允许运行一个任务，严禁多线程操作同一账号以规避风控。

#### 2.3.2 列表与监控

- **发布列表**：展示 Pending/Running/Failed 任务。
  - **操作**：支持暂停、取消、插队。
  - **监控**：实时进度条、当前执行步骤文字提示。
- **发布记录**：历史任务归档。
  - 提供多维度筛选（平台/账号/时间）。
  - 支持**一键重发**失败任务。

### 2.4 视频发布 (Video Publishing)

**核心功能**：配置并生成短视频发布任务。采用“账号驱动”的动态表单逻辑。

#### 2.4.1 发布配置流程

1.  **账号选择**：勾选一个或多个目标账号。
2.  **动态表单加载**：系统根据所选账号的平台属性，加载对应的表单项。
3.  **内容填写**：
    - **基础项**：视频上传、标题、描述、话题标签、封面图。
    - **平台特有项**：
      - **互动设置**：允许保存、谁可以看（公开/私密）。
      - **营销挂载**：购物车商品、团购链接、小程序（POI）。
      - **定时发布**：设置具体的发布时间点。
4.  **生成任务**：点击提交后，任务进入“发布管理”队列。

#### 2.4.2 批量视频发布

- **素材导入**：支持拖拽文件夹或多选文件导入。
- **变量支持**：标题/描述支持 `{文件名}`、`{序号}` 等占位符，实现批量发布的差异化。
- **参数同步**：支持“应用到全部”功能，快速统一配置。

#### 2.4.3 技术实现

- **执行引擎**：**Undetected Playwright (Headless Mode)**。
- **抗检测增强**：
  - **环境一致性**：强制使用与账号绑定的 `fingerprint_config.json`（UA、分辨率）。
  - **行为拟人**：内置随机操作延迟、非线性鼠标轨迹、Canvas 噪声注入。

### 2.5 图文发布 (Image Publishing)

**核心功能**：图文/图集内容的发布。

- **功能逻辑**：与视频发布一致，额外支持多图排序、图片编辑（裁剪/滤镜）、ALT 标签设置。
- **并发控制**：严格遵循全局配置的 `max_concurrent`，防止单 IP 并发过高。

### 2.6 数据中心 (Data Center)

- **数据抓取**：定期后台抓取播放量、点赞、评论、粉丝等核心指标。
- **可视化**：提供多维度数据看板（增长趋势图、爆款内容分析）。

### 2.7 评论及私信 (Interaction)

- **聚合管理**：统一展示所有账号的未读评论与私信。
- **快捷回复**：在软件内直接回复（通过后台接口或模拟操作），支持预设话术。

### 2.8 浏览器 (Browser Tool)

- **多开工具**：提供纯净的浏览器多开入口，用于人工干预或内容浏览。
- **环境对齐**：确保工具开启的浏览器与自动化发布使用完全相同的指纹环境。

### 2.9 文件管理 (File Management)

- **素材库**：视频/图片/文案的统一存储与分类。
- **去重检测**：发布前自动校验文件 MD5，提示潜在的重复发布风险。

### 2.10 设置 (Settings)

- **系统设置**：并发数限制、开机自启、关闭主面板行为（最小化/退出）。
- **网络设置**：全局代理配置、账号独立代理绑定。
- **数据维护**：缓存清理、数据备份与恢复。

## 3. 非功能性需求 (NFR)

1.  **抗检测性 (Anti-Detection)**
    - **指纹持久化**：账号生命周期内 UA 与 Canvas 指纹恒定。
    - **隐私保护**：WebRTC 默认走代理或禁用。
2.  **稳定性 (Stability)**
    - **僵尸进程治理**：软件退出或任务异常时，强制扫描并清理残留的 `chrome.exe` 进程。
    - **断点续传**：发布中断后支持从失败步骤重试。
3.  **安全性 (Security)**
    - Cookie 敏感数据必须加密存储（Fernet）。
    - 不上传任何用户凭证到云端。
