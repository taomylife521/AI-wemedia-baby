# 项目目录结构说明 (v2.0)

> **文档版本**：v2.0.1
> **最后更新**：2026-03-01
> **适用版本**：wemedia-baby v2.0+

本文档详细说明了媒小宝 (wemedia-baby) 项目的目录结构、关键文件及其用途，帮助开发者快速理解项目组织方式。

---

## 1. 根目录结构

```bash
wemedia-baby/
├── .venv/                  # [Python] 项目专用虚拟环境
├── config/                 # [Config] 全局配置文件与特性开关
├── data/                   # [Data] 运行时数据（数据库、Cookie、缓存）
├── docs/                   # [Docs] 项目文档（技术、实施、打包方案）
├── installers/             # [Build] 打包生成的安装程序
├── resources/              # [Res] 静态资源（图标、样式表）
├── scripts/                # [Scripts] 自动化脚本（打包、清理）
├── src/                    # [Source] 源代码核心目录
├── tests/                  # [Test] 单元测试与集成测试
├── tools/                  # [Tools] 外部依赖工具（FFmpeg, Updater）
├── main.py                 # [Entry] 应用程序统一入口
├── pyproject.toml          # [Conf] Python 项目配置 (Black, iSort)
├── requirements.txt        # [Dep] Python 依赖列表
├── README.md               # [Doc] 项目说明与快速开始
├── CHANGELOG.md            # [Doc] 变更日志
├── CONTRIBUTING.md         # [Doc] 贡献指南
└── .gitignore              # [Git] 版本控制忽略规则
```

---

## 2. 源代码目录 (`src/`)

采用核心 4 层 DDD (领域驱动设计) 架构：

```bash
src/
├── domain/                 # [领域层] 业务实体与防腐层接口
├── services/               # [服务层] 核心业务逻辑
│   ├── account/            # 账号管理服务
│   ├── browser/            # 浏览器控制服务 (Playwright)
│   ├── publish/            # 发布与任务调度服务
│   └── ...
├── infrastructure/         # [基础设施层] 底层技术实现
│   ├── storage/            # 数据库与文件存储
│   ├── network/            # 网络请求封装
│   ├── common/             # 通用组件 (DI, EventBus, Config)
│   └── ...
├── plugins/                # [插件层-开源] 标准平台插件
│   ├── douyin/             # 抖音插件实现
│   ├── kuaishou/           # 快手插件实现
│   └── plugin_manager.py   # v2.0 插件管理器
├── plugins_pro/            # [插件层-闭源] 高级平台插件 (Git Submodule)
│   └── (xiaohongshu, wechat_video...)
├── ui/                     # [界面层] MVVM 视图实现
│   ├── pages/              # 各功能页面 (Account, Publish...)
│   ├── components/         # 通用 UI 组件
│   ├── dialogs/            # 弹窗与向导
│   └── main_window.py      # 主窗口框架
└── utils/                  # [工具类] 通用辅助函数
```

---

## 3. 配置文件目录 (`config/`)

统一管理应用配置与功能特性：

```bash
config/
├── app_config.json         # 应用基础配置 (浏览器方案、默认设置)
├── feature_flags.py        # 特性开关 (控制开源/Pro功能、插件系统启用)
├── platforms/              # 平台相关的默认参数配置
└── selectors/              # 网页元素选择器配置
```

---

## 4. 关键文件说明

| 文件路径                                        | 说明                                                                        |
| :---------------------------------------------- | :-------------------------------------------------------------------------- |
| **main.py**                                     | **程序入口**。负责初始化 DI 容器、加载配置、启动插件系统、拉起 GUI。        |
| **src/plugins/plugin_manager.py**               | **插件管理器**。负责扫描 `src/plugins` 和 `src/plugins_pro`，加载平台插件。 |
| **config/feature_flags.py**                     | **功能开关**。控制 Pro 功能的启用/禁用，定义 `USE_PLUGIN_SYSTEM` 常量。     |
| **scripts/build_nuitka.ps1**                    | **打包脚本**。用于将项目编译为 exe 可执行文件，自动处理插件目录包含。       |
| **src/infrastructure/storage/database_init.py** | **数据库初始化**。首次运行时用于创建 SQLite 表结构。                        |

---

## 5. 打包与发布相关

```bash
scripts/
├── build_nuitka.ps1        # Nuitka 编译打包脚本 (推荐)
├── build_fast.ps1          # PyInstaller 快速打包脚本 (测试用)
└── clean_project.ps1       # 清理构建产物与临时文件
```

**输出目录**：

- `dist/`：存放最终打包生成的 `.exe` 文件或安装包。
- `build/`：存放打包过程中的临时编译文件 (已忽略)。

---

## 6. 数据存储 (`data/`)

> 注意：`data/` 目录默认被 git 忽略，仅保留空目录结构。

- `database.db`: SQLite 数据库文件 (用户信息、任务记录)。
- `cookies/`: 浏览器 Cookie 存储。
- `logs/`: 应用程序运行日志。
