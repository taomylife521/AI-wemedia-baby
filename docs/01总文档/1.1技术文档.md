## 1. 项目概述

### 1.1 软件简介

**媒小宝 (WeMediaBaby)** 是一款基于 **平台插件化架构** 的自媒体多账号管理与自动化发布工具。它采用“开源核心 + 闭源扩展”的混合部署模式，旨在为创作者提供安全、高效、可扩展的内容管理解决方案。

### 1.2 核心价值

- **插件化架构**：通过 `src/plugins` 接口层实现多平台隔离，支持开源社区贡献与商业插件的热插拔。
- **双核浏览器引擎**：
  - **内嵌模式** (`QWebEngine`): 提供轻量级的日常浏览与 Cookie 维护体验。
  - **自动化模式** (`PlaywrightService`): 提供隔离环境下的自动化操作（登录、发布），具备指纹混淆 (`Undetected`) 能力。
- **UI/逻辑解耦**：浏览器控制逻辑封装在服务层，通过信号与 UI (`BrowserControlDialog`) 通信，确保界面响应与业务逻辑分离。
- **混合部署**：核心框架开源 (AGPLv3)，高级商业插件 (Pro) 可动态加载 (`src/plugins_pro`)。

### 1.3 版本功能对比

| 功能模块       | 开源版 (Community) | 专业版 (Pro)     | 说明                              |
| :------------- | :----------------- | :--------------- | :-------------------------------- |
| **支持平台**   | 抖音, 快手         | + 小红书, 视频号 | 闭源插件存储于 `src/plugins_pro/` |
| **发布模式**   | 单任务发布         | 批量队列发布     | Pro 支持并发控制与定时任务        |
| **浏览器引擎** | 基础防检测         | 增强指纹混淆     | Pro 包含更严格的指纹一致性策略    |
| **账号管理**   | 本地加密存储       | + 云端备份       | Pro 支持跨设备同步 (开发中)       |

### 1.4 技术亮点

- **平台插件化**：基于 `Protocol` 定义标准接口 (`LoginPlugin`, `PublishPlugin`)，实现业务逻辑解耦。
- **UI/Service分离**：浏览器自动化逻辑 (`PlaywrightBrowserService`) 与界面 (`BrowserControlDialog`) 完全剥离，通过信号通信。
- **深度抗检测**：集成 `undetected-playwright`，动态注入 Canvas/WebGL 指纹，模拟真实用户行为。
- **纯异步架构**：全链路 `asyncio` + `qasync`，保证复杂自动化任务下的 UI 流畅度。
- **功能特性开关**：`FeatureFlags` 控制系统，支持细粒度的功能鉴权。

## 2. 系统架构

### 2.1 分层架构设计 (5层模型)

项目基于领域驱动设计 (DDD) 思想，增加了独立的 **插件层** 以支持扩展性。

```mermaid
graph TD
    subgraph "表现层 (Presentation Layer)"
        UI[PySide6 + FluentWidgets]
        Pages[Pages (Account, Publish)]
        Dialogs[Dialogs (BrowserControl)]
        UI --> Services
    end

    subgraph "应用层 (Application Layer)"
        Services[Services (Account, Publish, Batch)]
        BrowserService[PlaywrightBrowserService]
        Services --> Domain
        Services --> Plugins
    end

    subgraph "领域层 (Domain Layer)"
        Domain[Models (Account, Task)]
        Repos[Repositories Interface]
    end

    subgraph "插件层 (Plugin Layer)"
        PluginMgr[PluginManager]
        Interfaces[Interfaces (Login/Publish)]
        Impl_Open[Open Plugins (src/plugins)]
        Impl_Pro[Pro Plugins (src/plugins_pro)]
        PluginMgr --> Interfaces
        PluginMgr --> Impl_Open
        PluginMgr --> Impl_Pro
    end

    subgraph "基础设施层 (Infrastructure Layer)"
        Infra[Storage, Config, Logger]
        BrowserFactory[BrowserFactory]
        BrowserFactory --> Playwright
    end

    Services --> Infra
    Plugins --> Infra
```

### 3.2 开发规范

1.  **异步优先**：所有 I/O（数据库/网络/文件）必须使用 `async/await`，禁止阻塞 UI 线程。
2.  **类型注解**：所有函数签名必须包含 `typing` 类型提示。
3.  **依赖注入**：通过 `ServiceLocator` 获取服务实例，避免全局变量。
4.  **异常处理**：业务层抛出领域异常 (`AppError`)，UI 层统一用 `Toast/Dialog` 提示。
5.  **代码注释**：所有注释使用中文，便于团队协作。

## 4. 核心模块详解

### 4.1 账号管理模块 (`src/services/account`)

**职责**：账号的生命周期管理，包括登录、Cookie 持久化、状态监控。

| 类/文件           | 描述                                                                      |
| :---------------- | :------------------------------------------------------------------------ |
| `AccountService`  | 账号服务入口，提供 `add_account`, `verify_login`, `delete_account` 等方法 |
| `AccountVerifier` | 账号有效性验证器，检测 Cookie 是否过期、账号是否被封禁                    |
| `CookieManager`   | Cookie 加解密与持久化，存储于 `%APPDATA%/WeMediaBaby/data/cookies/`       |

**数据标识变迁**：

- **v1.1.0 变更**：移除了冗余的 `account_name` (备注名) 字段，统一使用 `platform_username` (平台唯一用户名，如抖音号) 作为账号的底层目录标识，与 `platform` 共同构成数据存储路径。UI显示的名称可能依然是用户昵称，但底层目录强绑定 `platform_username`。

**安全机制**：

- 加密密钥存储在系统 `Keyring` 中。
- 账号数据存储于 `%APPDATA%`，实现与源码目录的物理隔离。

### 4.2 发布模块 (`src/services/publish` + `src/infrastructure/common/pipeline`)

**职责**：编排内容发布流程，支持单发和批量发布。

**核心设计**：采用"管道-过滤器 (Pipeline & Filter)"模式。

```
PublishRequest → [Filter1] → [Filter2] → ... → [FilterN] → PublishResult

Filter 链示例:
  1. PermissionFilter: 权限校验
  2. MediaValidateFilter: 素材格式校验
  3. AccountLoadFilter: 加载账号 Cookie
  4. PlatformPublishFilter: 调用平台插件执行上传
  5. RecordSaveFilter: 保存发布记录
```

**并发控制**：使用 `asyncio.Semaphore` 限制全局并发数，避免触发平台风控。

**断点续传**：发布状态实时写入 SQLite，支持异常后恢复。

### 4.3 浏览器架构 (`src/infrastructure/browser` + `src/services/browser`)

**核心理念**：UI与逻辑分离 (Decoupled UI & Logic)，支持多引擎切换。

**架构组件**：

1.  **服务层 (`PlaywrightBrowserService`)**：
    - **职责**：纯业务逻辑，不依赖 GUI。负责浏览器的生命周期管理、Cookie 注入/提取、登录状态监控。
    - **通信**：通过 PyQt 信号 (`Signal`) 广播事件（如 `browser_launched`, `account_saved`），实现与 UI 的完全解耦。

2.  **UI 层 (`BrowserControlDialog`)**：
    - **职责**：独立的可复用组件，仅负责显示浏览器状态（如"等待登录"）和提供手动操作入口（"手动保存"）。
    - **交互**：监听 Service 信号更新界面，不直接操作底层浏览器对象。

3.  **基础设施层 (`BrowserFactory` & `UndetectedBrowserManager`)**：
    - **`BrowserFactory`**：工厂模式，根据配置 (`browser_scheme`) 实例化对应的底层管理器。
    - **`UndetectedBrowserManager`**：封装 `playwright` 与 `undetected-playwright`，负责底层的指纹混淆、Context 创建与进程管理。

**双引擎模式**：

| 引擎类型     | 技术栈                                 | 适用场景                                 | 优点                     | 缺点                       |
| :----------- | :------------------------------------- | :--------------------------------------- | :----------------------- | :------------------------- |
| **内嵌引擎** | `QWebEngineView` (Chromium)            | 日常浏览、快速检查、Cookie 维护          | UI 集成度高，资源占用低  | 自动化能力弱，抗检测性差   |
| **外置引擎** | `Playwright` / `Undetected-Playwright` | 批量登录、复杂自动化任务、高强度风控对抗 | 自动化能力强，抗检测性好 | 资源占用高，需弹出独立窗口 |

**抗检测技术 (Undetected Playwright)**：

- **指纹一致性**：强制锁定 `fingerprint_config.json`，确保 Canvas/WebGL 指纹在同账号下始终一致。
- **特征注入**：动态注入 JS 脚本还原真实浏览器特征 (Navigator, Screen, AudioContext)。
- **环境隔离**：每个账号拥有独立的 `storage_state.json` (Cookie/LocalStorage) 和用户目录。

### 4.4 数据存储 (`src/infrastructure/storage`)

**职责**：SQLite 数据库的异步访问。

| 类/文件            | 描述                             |
| :----------------- | :------------------------------- |
| `database.py`      | 数据库连接池与 CRUD 封装         |
| `database_init.py` | 数据库初始化脚本，创建所有表结构 |

**性能优化**：

- 启用 `WAL` (Write-Ahead Logging) 模式。
- 设置 `synchronous=NORMAL` 平衡性能与安全。
- 增加缓存至 10MB (`cache_size=-10000`)。

### 4.5 UI 层 (`src/ui`)

**职责**：使用 PySide6 + Fluent-Widgets 构建现代化界面。

**架构模式**：MVVM (Model-View-ViewModel)

| 目录          | 描述                                                                                         |
| :------------ | :------------------------------------------------------------------------------------------- |
| `pages/`      | 页面模块：工作台 (`workspace_page`)、账号列表 (`account_list_page`)、发布页 (`publish_page`) |
| `dialogs/`    | 弹窗：登录 (`login_dialog`)、新建任务 (`create_task_dialog`)                                 |
| `viewmodels/` | ViewModel：封装业务逻辑，暴露数据给 View                                                     |
| `components/` | 通用组件：账号卡片、进度条、表格                                                             |

**异步集成**：通过 `qasync` 将 `asyncio` 协程融入 Qt 事件循环，避免 UI 卡顿。

---

### 4.6 插件系统 (`src/plugins` & `plugins_pro/`)

**设计目标**：实现多平台接入的标准化与扩展性，隔离开源框架与闭源商用逻辑。

**核心接口** (`src/plugins/interfaces`)：

- `LoginPlugin`：定义标准登录流程（Login URL, 验证码检测, UserInfo 提取）。
- `PublishPlugin`：定义标准发布流程（Video Upload, Content Filling, Final Publish）。

**双仓库加载机制**：

- **开源插件**：自动扫描 `src/plugins/`（如 Douyin, Kuaishou）。
- **闭源插件**：自动扫描 `src/plugins_pro/`（如 Xiaohongshu, WeChat Video）。
- `PluginManager` 负责根据 `platform` 键值动态加载对应模块。

**架构优势**：

- **热插拔**：新增平台无需修改主程序代码。
- **配置分离**：配合 `config.json` 与 `selectors.py`，将易变的页面元素定位逻辑抽离，支持快速热修。

## 5. 配置文件说明

### 5.1 `config/app_config.json`

```json
{
  "browser_scheme": "mixed", // 浏览器方案: mixed (内嵌+外置), playwright (仅外置), undetected_playwright (抗检测外置)
  "max_concurrent": 3, // 最大并发发布数
  "auto_update": true // 是否自动检查更新
}
```

### 5.2 `config/platforms/*.json`

平台专属配置，包括 URL、Cookie 域、发布流程参数等。

### 5.3 `config/selectors/*.json`

CSS/XPath 选择器清单，用于定位平台页面元素，支持远程热更新。

## 6. 构建与部署

```powershell

python -m venv .venv
```

## 7. 安全说明

1.  **代码保护**：生产版本使用 Nuitka 编译为 C++ 机器码，防止反编译。
2.  **数据加密**：所有敏感数据 (Cookie, Token) 使用 `Fernet` 加密后落盘。
3.  **密钥管理**：加密密钥存储在操作系统凭据库，不以明文形式存在于文件系统。
4.  **隐私保护**：所有用户数据本地存储，不经过云端中转。

## 8. 功能开关系统

### 8.1 概述

功能开关系统 (`config/feature_flags.py`) 用于控制 Community 和 Pro 版本的功能可用性。

### 8.2 核心类

| 类/函数                    | 描述                                |
| :------------------------- | :---------------------------------- |
| `FeatureFlags`             | 功能开关管理器，定义开源/Pro 功能集 |
| `FeatureNotAvailableError` | 功能不可用异常                      |
| `@require_pro`             | Pro 版本要求装饰器                  |
| `@require_feature`         | 指定功能要求装饰器                  |
| `@require_platform`        | 指定平台要求装饰器                  |

### 8.3 使用示例

```python
from config.feature_flags import FeatureFlags, require_pro

# 检查功能是否启用
if FeatureFlags.is_feature_enabled('batch_publish'):
    # 执行批量发布逻辑
    pass

# 装饰器保护 Pro 功能
@require_pro
def pro_only_function():
    pass
```

### 8.4 闭源模块标识

所有闭源模块在文件头部包含 `# PRO_ONLY` 注释，完整清单见 `.opensource-ignore` 文件。
