# 双仓库实施指南 (Submodule 方案)

> **文档版本**：v1.0.1
> **最后更新**：2026-03-01
> **适用场景**：开源主仓库 + 闭源插件仓库

本文档指导如何将 `wemedia-baby` 拆分为两个独立的 Git 仓库进行管理。

---

## 1. 架构说明

- **主仓库 (Public)**：包含核心框架、UI、以及开源插件 (`src/plugins/`)。
- **子仓库 (Private)**：包含 Pro 版插件 (`src/plugins_pro/`)，作为 Git Submodule 挂载。

## 2. 准备工作

请确保你拥有两个 Git 仓库地址：

1.  **主仓库 URL** (例如 GitHub): `git@github.com:username/wemedia-baby.git`
2.  **子仓库 URL** (例如 Gitee/GitLab): `git@gitee.com:username/wemedia-baby-pro.git`

---

## 3. 首次拆分步骤 (For Maintainer)

如果你是项目维护者，需要首次执行拆分操作：

### 3.1 准备子仓库代码

1.  将现有的 `src/plugins_pro` 目录移动到临时位置（例如桌面）。
2.  在临时位置初始化 Git 仓库并推送：
    ```bash
    cd /path/to/desktop/plugins_pro
    git init
    git add .
    git commit -m "init: pro plugins v1.0"
    git branch -M main
    git remote add origin <子仓库 URL>
    git push -u origin main
    ```

### 3.2 配置主仓库

1.  回到项目根目录，确保 `src/plugins_pro` 文件夹已清空/删除。
2.  添加 Submodule：

    ```bash
    # 将子仓库挂载到 src/plugins_pro
    git submodule add <子仓库 URL> src/plugins_pro
    ```

    > 执行后，`src/plugins_pro` 下会出现子仓库文件，根目录会生成 `.gitmodules` 文件。

3.  提交主仓库更改：
    ```bash
    git add .gitmodules src/plugins_pro
    git commit -m "chore: add pro plugins submodule"
    git push origin main
    ```

---

## 4. 开发与使用 (For Setup)

### 4.1 克隆完整项目 (含 Pro 插件)

```bash
# 方式一：递归克隆 (推荐)
git clone --recursive <主仓库 URL>

# 方式二：分步克隆
git clone <主仓库 URL> wemedia-baby
cd wemedia-baby
git submodule init
git submodule update
```

### 4.2 仅克隆开源版

```bash
git clone <主仓库 URL>
# 不执行 git submodule init 即可，程序会自动检测并以开源模式运行。
```

### 4.3 更新 Submodule

当 Pro 插件有更新时：

```bash
git submodule update --remote src/plugins_pro
```
