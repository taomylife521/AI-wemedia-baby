# 双仓库实施指南 (Submodule 方案)

> **文档版本**：v2.0.0
> **最后更新**：2026-03-03
> **适用场景**：开源主仓库 + 闭源 Pro 仓库

---

## 1. 架构说明

项目采用 **主仓库 + 私有子仓库** 的双仓库模式：

| 仓库           | 类型    | 地址                                                 | 说明                   |
| -------------- | ------- | ---------------------------------------------------- | ---------------------- |
| **主仓库**     | Public  | `https://github.com/chitang818/wemedia-baby.git`     | 核心框架、UI、社区插件 |
| **Pro 子仓库** | Private | `https://github.com/chitang818/wemedia-baby-pro.git` | Pro 插件 & Pro 功能    |

### 1.1 Pro 子仓库包含的闭源内容

Pro 私有仓库通过 Git Submodule 挂载到主仓库的以下两个目录：

```
wemedia-baby/
├── src/
│   ├── plugins/
│   │   ├── community/          # ✅ 开源 - 社区插件 (抖音等)
│   │   ├── core/               # ✅ 开源 - 插件核心框架
│   │   └── pro/                # 🔒 闭源 - Pro 平台插件 (Submodule)
│   │       ├── wechat_video/   #    微信视频号插件
│   │       └── xiaohongshu/    #    小红书插件
│   └── pro_features/           # 🔒 闭源 - Pro 增值功能 (Submodule)
│       ├── batch/              #    批量发布功能
│       ├── data_center/        #    数据中心功能
│       └── interaction/        #    互动管理功能
```

### 1.2 隔离机制

- **`.gitignore`** 中已配置 `src/plugins/pro/` 和 `src/pro_features/`，确保这两个目录不会被主仓库追踪。
- **`.gitmodules`** 中声明了两个 Submodule，均指向同一个私有仓库 `wemedia-baby-pro`。

---

## 2. 首次拆分步骤 (维护者操作)

### 2.1 准备 Pro 子仓库

1. 在 GitHub 上创建私有仓库 `wemedia-baby-pro`（如果尚未创建）。

2. 在本地新建一个临时目录，将两个 Pro 目录合并进去：

   ```bash
   # 创建临时工作区
   mkdir ~/wemedia-baby-pro-temp
   cd ~/wemedia-baby-pro-temp
   git init

   # 复制 Pro 插件和 Pro 功能
   cp -r /path/to/wemedia-baby/src/plugins/pro ./plugins_pro
   cp -r /path/to/wemedia-baby/src/pro_features ./pro_features

   # 提交并推送
   git add .
   git commit -m "init: Pro 插件与 Pro 功能 v1.0.0"
   git branch -M main
   git remote add origin https://github.com/chitang818/wemedia-baby-pro.git
   git push -u origin main
   ```

### 2.2 在主仓库中配置 Submodule

1. 删除主仓库中已有的 Pro 目录（先确保已备份到 Pro 子仓库）：

   ```bash
   cd /path/to/wemedia-baby
   rm -rf src/plugins/pro
   rm -rf src/pro_features
   ```

2. 添加 Submodule 挂载：

   ```bash
   git submodule add https://github.com/chitang818/wemedia-baby-pro.git src/plugins/pro
   git submodule add https://github.com/chitang818/wemedia-baby-pro.git src/pro_features
   ```

3. 提交主仓库：

   ```bash
   git add .gitmodules src/plugins/pro src/pro_features
   git commit -m "chore: 添加 Pro 插件与 Pro 功能 Submodule"
   git push origin main
   ```

## 3. 开发与使用

### 3.1 克隆完整项目 (含 Pro 插件)

```bash
# 方式一：递归克隆（推荐）
git clone --recursive https://github.com/chitang818/wemedia-baby.git

# 方式二：分步克隆
git clone https://github.com/chitang818/wemedia-baby.git
cd wemedia-baby
git submodule init
git submodule update
```

### 3.2 仅克隆开源版

```bash
git clone https://github.com/chitang818/wemedia-baby.git
# 不执行 git submodule init，程序会自动检测并以开源模式运行
```

### 3.3 更新 Submodule

当 Pro 插件或功能有更新时：

```bash
git submodule update --remote src/plugins/pro
git submodule update --remote src/pro_features
```

## 4. 注意事项

> [!IMPORTANT]
> Pro 子仓库为 **Private 私有仓库**，克隆时需要有 GitHub 访问权限。没有权限的用户只能使用开源社区版。

> [!WARNING]
> 推送代码前，请务必确认 `src/plugins/pro/` 和 `src/pro_features/` 没有被意外添加到主仓库的 Git 追踪中。可通过 `git status` 检查。
