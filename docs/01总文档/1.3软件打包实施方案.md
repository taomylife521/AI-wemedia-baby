> **文档版本**：v2.0.1

> **当前软件版本**：v1.0.1
> **状态**：正式执行版

---

## 1. 方案概述

本项目采用 **双轨制打包策略**，以平衡开发效率与发布质量：

| 阶段       | 适用场景                    | 核心工具                         | 特点                                                     | 产物路径                 |
| :--------- | :-------------------------- | :------------------------------- | :------------------------------------------------------- | :----------------------- |
| **迭代期** | 内部测试、快速验证、QA 回归 | **PyInstaller** + **Inno Setup** | 🚀 极速构建 (<2分钟) ‧ 📦 调试友好 ‧ 🔓 代码无混淆       | `dist/fast/WeMediaBaby/` |
| **发布期** | 正式发版、交付客户          | **Nuitka** + **Inno Setup**      | 🛡️ 高安全 (转C++编译) ‧ ⚡ 高性能 (启动快) ‧ 🔒 防反编译 | `dist/secure/main.dist/` |

两种方案均通过 **Inno Setup** 生成 Windows 安装包，安装包输出到 `dist/installers/` 目录。

## 3. 版本管理

### 3.1 版本号来源

项目采用 **单一权威版本源 (`src/version.py`)** 策略，构建脚本在运行时自动读取此文件中的版本号。

| 文件               | 用途                       | 由谁维护            |
| :----------------- | :------------------------- | :------------------ |
| `src/version.py`   | 运行时版本号（程序内读取） | `update_version.py` |
| `pyproject.toml`   | 项目元数据版本号           | `update_version.py` |
| `setup_script.iss` | 安装包版本号               | `update_version.py` |
| `CHANGELOG.md`     | 变更日志                   | `update_version.py` |

### 3.2 更新版本号

**方式一：使用交互菜单** (推荐)

```batch
# 从项目根目录双击或运行
version.bat
```

按提示选择更新类型：

- `[1]` 补丁更新 (Patch)：`1.0.1` → `1.0.2`
- `[2]` 次版本更新 (Minor)：`1.0.1` → `1.1.0`
- `[3]` 主版本更新 (Major)：`1.0.1` → `2.0.0`

**方式二：命令行直接调用**

```bash
python scripts/release/update_version.py patch   # 补丁更新
python scripts/release/update_version.py minor   # 次版本更新
python scripts/release/update_version.py major   # 主版本更新
```

## 4. 迭代期方案 (PyInstaller + Inno Setup)

### 4.1 一键构建

`build_fast.ps1` 脚本已集成 **PyInstaller 构建 + Inno Setup 打包** 完整流程，一条命令即可完成：

```powershell
# 方式一：直接运行脚本
.\scripts\build\build_fast.ps1

# 方式二：通过交互菜单
.\build.bat  →  选 [1]
```

**脚本执行流程**：

1. 从 `src/version.py` 读取版本号
2. 检测 / 激活虚拟环境 (`.venv`)
3. 检查 / 安装 PyInstaller 依赖
4. 清理旧构建产物 (`dist/fast/`)
5. 执行 `PyInstaller` 构建 → 生成可执行文件到 `dist/fast/WeMediaBaby/`
6. 复制附加文件 (README、调试脚本)
7. 自动检测 **Inno Setup** 编译器 → 编译安装包到 `dist/installers/`

### 4.2 构建产物

| 产物              | 路径                                                   |
| :---------------- | :----------------------------------------------------- |
| 可执行程序目录    | `dist/fast/WeMediaBaby/`                               |
| 调试启动脚本      | `dist/fast/WeMediaBaby/debug_start.bat`                |
| 安装包 (自动生成) | `dist/installers/WeMediaBaby_Setup_Fast_v[版本号].exe` |

### 4.3 PyInstaller 配置说明 (`main.spec`)

打包配置文件位于 `scripts/build/main.spec`，关键设置：

- **入口文件**：`main.py` (项目根目录)
- **打包模式**：目录模式 (非单文件)，使用 UPX 压缩
- **隐藏导入**：`qasync`、`pyside6_fluent_widgets`、`playwright`、`undetected_playwright`、`dns` 等
- **窗口模式**：无控制台窗口 (`console=False`)
- **应用图标**：`resources/icons/app.ico`

## 5. 发布期方案 (Nuitka + Inno Setup)

### 5.1 执行构建

```powershell
# 方式一：直接运行脚本
$env:NUITKA_AssumeYesForDownloads = 1   # 首次运行需要
.\scripts\build\build_nuitka.ps1

# 方式二：调试模式 (保留控制台窗口)
.\scripts\build\build_nuitka.ps1 -Debug

# 方式三：仅清理构建产物
.\scripts\build\build_nuitka.ps1 -CleanOnly

# 方式四：通过交互菜单
.\build.bat  →  选 [2]
```

### 5.2 Nuitka 构建配置

| 配置项           | 值                                                  |
| :--------------- | :-------------------------------------------------- |
| 编译模式         | `--standalone` (独立可执行目录)                     |
| 输出目录         | `dist/secure/`                                      |
| Qt 插件          | `pyside6` (自动启用)                                |
| Qt 子插件        | `sensible`, `styles`, `iconengines`, `imageformats` |
| 链接时优化 (LTO) | 关闭 (减少首次编译时间)                             |
| 控制台窗口       | 默认禁用 (发布模式)，`-Debug` 参数可启用            |

### 5.3 构建产物

| 产物              | 路径                                   |
| :---------------- | :------------------------------------- |
| 可执行程序目录    | `dist/secure/main.py.dist/`            |
| 安装包 (自动生成) | `dist/installers/` (需安装 Inno Setup) |

## 9. 常见问题 (FAQ)

**Q: `build_fast.ps1` 执行后提示语法错误？**
A: 请确认 PowerShell 版本 ≥ 5.1，并以 `powershell -ExecutionPolicy Bypass -File scripts\build\build_fast.ps1` 方式运行。

**Q: Inno Setup 编译报错 "Source file not found"？**
A: 请先运行 `.\scripts\build\build_fast.ps1` 生成源文件到 `dist/fast/WeMediaBaby/` 目录。

**Q: Nuitka 编译极慢？**
A: 首次编译需要分析依赖并下载 C 编译器，后续会利用缓存加速。建议关闭杀毒软件并确保磁盘 I/O 性能。

**Q: 编译时提示找不到 `src/pro_features`？**
A: 该目录为可选的 Pro 版付费功能模块。如果是开源版构建，此警告可安全忽略。如需包含，请先初始化 Git Submodule：`git submodule update --init --recursive`。

**Q: 安装包安装后无法启动？**
A: 请检查目标机器是否已安装 VC++ 运行时库。Nuitka 构建通常自动处理此依赖，也可手动安装 `VC_redist.x64.exe`。
