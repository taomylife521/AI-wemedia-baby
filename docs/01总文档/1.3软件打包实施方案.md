# 媒小宝 (wemedia-baby) 软件打包实施方案

> **文档版本**：v1.2.0
> **最后更新**：2026-02-09
> **状态**：正式执行版 (适配 v2.0 插件架构)

## 1. 方案概述

本项目采用 **双轨制打包策略**，以平衡开发效率与发布质量：

| 阶段       | 场景                       | 核心工具组合                        | 特点                                                                                          | 产物格式                        |
| :--------- | :------------------------- | :---------------------------------- | :-------------------------------------------------------------------------------------------- | :------------------------------ |
| **迭代期** | 内部测试、QA验证、快速体验 | **PyInstaller** + **Inno Setup**    | 🚀 **极速构建** (构建<2分钟，安装包<1分钟)<br>📦 便于排查文件缺失问题<br>🔓 代码无混淆保护    | `.exe` 安装包 (极速版)          |
| **发布期** | 正式发版、交付客户         | **Nuitka** + **Advanced Installer** | 🛡️ **高安全** (转C++编译，防反编译)<br>⚡ **高性能** (启动快)<br>🎨 **专业UI** (安装界面美观) | `.exe` / `.msi` 安装包 (专业版) |

## 2. 迭代期方案 (PyInstaller + Inno Setup)

### 2.1 第一步：构建可执行文件 (PyInstaller)

运行 PowerShell 脚本进行快速打包：

```powershell
.\scripts\build_fast.ps1
```

- **输出目录**：`dist/fast/WeMediaBaby`
- **调试说明**：如果程序启动失败，直接双击目录下的 `WeMediaBaby.exe`，此时会通过控制台输出错误日志。

### 2.2 第二步：制作安装包 (Inno Setup)

使用 Inno Setup 编译器将上述输出目录封装为安装程序。

1.  确保已安装 [Inno Setup 6.x](https://jrsoftware.org/isdl.php)。
2.  打开 `installers/inno_setup/setup_script.iss` 文件。
3.  点击 **Compile** 按钮。
4.  **输出产物**：`dist/installers/WeMediaBaby_Setup_Fast_v[版本号].exe`

## 3. 发布期方案 (Nuitka + Advanced Installer)

### 3.1 第一步：编译核心程序 (Nuitka)

运行 Nuitka 构建脚本（耗时较长，建议在高性能机器上运行）：

```powershell
# 首次运行
$env:NUITKA_AssumeYesForDownloads = 1
.\scripts\build_nuitka.ps1
```

- **输出目录**：`dist/secure/main.dist`
- **特性**：Python 代码被编译为机器码，极难还原。

### 3.2 第二步：制作专业安装包 (Advanced Installer)

使用 Advanced Installer 制作发布级安装包。

1.  打开 `installers/advanced_installer/WeMediaBaby.aip` (工程文件)。
2.  确保 **Files and Folders** 指向 `dist/secure/main.dist`。
3.  构建项目 (Build)。
4.  **输出产物**：`dist/installers/WeMediaBaby_Setup_Full_v[版本号].exe`

## 4. 资源文件清单 (通用)

无论使用哪种打包方式，以下目录和文件均会被自动包含：

1.  `config/`: 配置文件模板。
2.  `data/`: 初始数据目录结构。
3.  `src/infrastructure/plugins/`: 内置/旧版插件 (Legacy)。
4.  `src/plugins/`: **开源插件 (v2.0)** (必须包含)。
5.  `src/plugins_pro/`: **闭源插件 (v2.0)** (仅 Pro 版包含，需初始化 Submodule)。
6.  `resources/`: 图标、样式表。
7.  `tools/`: ffmpeg、updater 等外部工具。

**关于 Playwright 浏览器内核**：
打包程序默认 **不包含** 浏览器内核。程序首次运行或检测到缺失时，会提示用户下载或自动下载。

## 5. 常见问题 (FAQ)

**Q: Inno Setup 编译报错 "Source file not found"?**
A: 请先运行 `.\scripts\build_fast.ps1` 生成源文件。

**Q: Nuitka 编译极慢？**
A: 首次编译需要分析依赖并下载 C 编译器，后续会利用缓存加速。建议关闭杀毒软件。

**Q: 编译时提示找不到 `src/plugins_pro`？**
A: 请检查是否已初始化 Git Submodule：`git submodule update --init --recursive`。如果是开源版构建，该警告可忽略，但需确保代码中处理了目录缺失的情况。

**Q: 安装包安装后无法启动？**
A: 请检查是否缺少 VC++ 运行时库（通常 Nuitka 会自动处理，可以尝试在目标机器手动安装 `VC_redist.x64.exe`）。
